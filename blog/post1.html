Automate FastAPI Deployment with GitHub Actions

Learn how to deploy a FastAPI application using GitHub Actions for CI/CD automation.

  1. 1️. Set Up Your FastAPI Project

    Ensure your project has a main.py file and a requirements.txt for dependencies.

  2. 2️. Create a Dockerfile

    Define how your FastAPI app should be containerized.

    FROM python:3.9
WORKDIR /app
COPY . .
RUN pip install --no-cache-dir -r requirements.txt
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]

  3. 3️. Configure GitHub Actions

    Create a .github/workflows/deploy.yml file.

    name: FastAPI CI/CD

on: [push]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout Repository
        uses: actions/checkout@v2
      
      - name: Set up Docker
        uses: docker/setup-buildx-action@v1
      
      - name: Login to Docker Hub
        run: echo "${{ secrets.DOCKER_PASSWORD }}" | docker login -u "${{ secrets.DOCKER_USERNAME }}" --password-stdin
      
      - name: Build and Push Docker Image
        run: |
          docker build -t ${{ secrets.DOCKER_USERNAME }}/fastapi-app:latest .
          docker push ${{ secrets.DOCKER_USERNAME }}/fastapi-app:latest

  4. 4️. Deploy to Server

    Use SSH to pull the image and restart the container on your VPS.

    ssh user@your-server "docker pull ${{ secrets.DOCKER_USERNAME }}/fastapi-app:latest && docker run -d -p 8000:8000 --name fastapi-app ${{ secrets.DOCKER_USERNAME }}/fastapi-app:latest"
